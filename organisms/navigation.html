Navigation

사용자가 애플리케이션 내에서 이동할 수 있도록 도와주는 네비게이션 패턴 모음입니다.

Navigation Types

용도와 화면 크기에 따라 다양한 네비게이션 패턴을 사용할 수 있습니다.

Horizontal Nav

헤더에 위치하는 수평 네비게이션. 주요 섹션 이동에 적합

Vertical Nav

사이드바 형태의 수직 네비게이션. 많은 메뉴 항목에 적합

Bottom Nav

모바일 하단 고정 네비게이션. 터치 접근성이 좋음

Mobile Slide-out Menu

모바일에서 햄버거 메뉴 클릭 시 슬라이드되어 나타나는 네비게이션입니다.

Structure 
<!-- Overlay -->
<div class="fixed inset-0 z-50">
  <div class="fixed inset-0 bg-black/50" onclick="closeMenu()"></div>

  <!-- Slide-out panel -->
  <aside class="fixed left-0 top-0 bottom-0 w-72
    bg-white transform transition-transform duration-300
    -translate-x-full"> <!-- Closed state -->

    <!-- Header with close button -->
    <div class="h-14 px-4 border-b flex items-center justify-between">
      <span>Menu</span>
      <button onclick="closeMenu()">×</button>
    </div>

    <!-- Navigation links -->
    <nav class="p-4">...</nav>
  </aside>
</div>

/* Open state: remove -translate-x-full or add translate-x-0 */

Responsive Pattern

화면 크기에 따라 네비게이션 형태가 변하는 반응형 패턴입니다.

반응형 네비게이션 전략

  • • Desktop (lg+): 수평 네비게이션 또는 사이드바
  • • Tablet (md): 축소된 아이콘 사이드바 또는 햄버거 메뉴
  • • Mobile (sm-): 하단 네비게이션 또는 슬라이드 메뉴
Responsive Classes 
/* Desktop: 수평 네비게이션 표시 */
<nav class="hidden lg:flex items-center gap-4">
  <!-- Horizontal links -->
</nav>

/* Mobile: 햄버거 버튼 표시 */
<button class="lg:hidden">
  <!-- Hamburger icon -->
</button>

/* Mobile: 하단 네비게이션 */
<nav class="lg:hidden fixed bottom-0 left-0 right-0">
  <!-- Bottom nav items -->
</nav>

/* Desktop: 사이드바 */
<aside class="hidden lg:block w-64">
  <!-- Sidebar content -->
</aside>

Usage Guidelines

Do

  • • 현재 위치를 명확하게 표시하세요
  • • 메뉴 항목은 5-7개 이내로 유지하세요
  • • 모바일에서는 터치하기 쉬운 크기로 만드세요 (최소 44px)
  • • 중요한 항목을 먼저 배치하세요
  • • 아이콘과 텍스트를 함께 사용하여 명확성을 높이세요
  • • 키보드 네비게이션을 지원하세요

Don't

  • • 3단계 이상의 깊은 메뉴 구조를 만들지 마세요
  • • 데스크톱 네비게이션을 모바일에 그대로 사용하지 마세요
  • • 너무 많은 항목을 한 레벨에 넣지 마세요
  • • 아이콘만으로 네비게이션을 구성하지 마세요
  • • 스크롤이 필요한 긴 수평 네비게이션을 만들지 마세요
  • • 드롭다운 안에 또 드롭다운을 넣지 마세요
