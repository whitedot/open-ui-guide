Home Organisms Header

Header

페이지 상단에 위치하여 브랜드 아이덴티티, 네비게이션, 사용자 액션을 제공하는 컴포넌트입니다.

Anatomy

Header는 일반적으로 3개의 영역으로 구성됩니다.

1. Logo / Brand
2. Navigation (optional)
3. Actions / User

1. Logo / Brand

로고, 브랜드명, 홈 링크

2. Navigation

주요 메뉴 링크 (선택적)

3. Actions / User

검색, 알림, 프로필, 로그인

Basic Header

가장 기본적인 헤더 형태입니다. 로고와 주요 액션만 포함합니다.

Brand
HTML 
<header class="h-16 flex items-center justify-between px-6 bg-white dark:bg-gray-800 border-b border-gray-200 dark:border-gray-700">
  <!-- Logo -->
  <a href="/" class="flex items-center gap-3">
    <div class="w-8 h-8 bg-violet-600 rounded-lg">...</div>
    <span class="text-lg font-bold text-gray-900 dark:text-white">Brand</span>
  </a>

  <!-- Actions -->
  <div class="flex items-center gap-2">
    <button class="h-10 px-4 text-gray-600 hover:bg-gray-100 ...">로그인</button>
    <button class="h-10 px-4 bg-gray-600 hover:bg-gray-700 text-white ...">시작하기</button>
  </div>
</header>

Sticky Header

스크롤해도 상단에 고정되는 헤더입니다.

fixed vs sticky

  • • fixed: 뷰포트 기준으로 고정, 부모 요소의 레이아웃에서 제외됨
  • • sticky: 스크롤 위치에 따라 상대적으로 고정, 레이아웃 공간 유지
CSS Classes 
/* Fixed Header (뷰포트 상단 고정) */
<header class="fixed top-0 left-0 right-0 z-50 h-16 ...">
/* body에 padding-top 필요: pt-16 */

/* Sticky Header (스크롤 시 상단 고정) */
<header class="sticky top-0 z-50 h-16 ...">

/* Sticky with backdrop blur */
<header class="sticky top-0 z-50 h-16 bg-white/80 backdrop-blur-lg ...">

Usage Guidelines

Do

  • • 로고는 항상 홈페이지로 링크하세요
  • • 모바일에서는 h-14, 데스크톱에서는 h-16을 권장합니다
  • • 고정 헤더 사용 시 body에 padding-top을 추가하세요
  • • 주요 네비게이션은 5개 이하로 유지하세요
  • • 알림 배지는 읽지 않은 항목이 있을 때만 표시하세요
  • • z-index는 다른 요소보다 높게 설정하세요 (z-50)

Don't

  • • 헤더 높이를 너무 크게 설정하지 마세요 (80px 초과)
  • • 모바일에서 복잡한 네비게이션을 표시하지 마세요
  • • 투명 헤더에 어두운 배경을 사용하지 마세요
  • • 여러 개의 Primary 버튼을 헤더에 넣지 마세요
  • • 로고 없이 헤더를 구성하지 마세요
  • • 스크롤 시 헤더가 콘텐츠를 가리지 않도록 하세요
