Input

사용자로부터 텍스트 데이터를 입력받는 폼 요소입니다. 다양한 타입과 상태를 제공합니다.

Input Types

다양한 데이터 타입에 맞는 입력 필드입니다. 적절한 타입을 사용하면 모바일에서 최적화된 키보드가 표시됩니다.

HTML 
<!-- Label + Input 기본 패턴 -->
<label class="input-label">Text</label>
<input type="text" placeholder="..." class="input input-md input-full">

<!-- 다른 타입들 -->
<input type="email" class="input input-md input-full">
<input type="password" class="input input-md input-full">
<input type="number" class="input input-md input-full">
<input type="tel" class="input input-md input-full">
<input type="url" class="input input-md input-full">
<input type="search" class="input input-md input-full">
<input type="date" class="input input-md input-full">

States

입력 필드의 상호작용 상태입니다.

CSS Classes 
/* Hover */
hover:border-gray-400 dark:hover:border-gray-500

/* Focus */
focus:ring-2 focus:ring-violet-500 focus:border-transparent

/* Disabled */
disabled:opacity-50 disabled:cursor-not-allowed disabled:bg-gray-50

/* Read-only */
read-only:bg-gray-50 read-only:cursor-default

Validation States

폼 유효성 검사 결과를 시각적으로 표시합니다.

필수 항목입니다

확인되었습니다

보안이 취약할 수 있습니다

HTML 
<!-- Error State -->
<input class="input input-md input-error input-full">
<p class="input-helper-error">에러 메시지</p>

<!-- Success State -->
<input class="input input-md input-success input-full">
<p class="input-helper-success">성공 메시지</p>

<!-- Warning State -->
<input class="input input-md input-warning input-full">

Labels & Helper Text

레이블은 입력 필드의 목적을 설명하는 필수 요소입니다. for 속성으로 입력 필드와 연결하여 접근성을 높입니다.

Basic Labels

8자 이상, 영문/숫자/특수문자 포함

HTML 
<!-- Basic Label -->
<label for="input-id" class="input-label">이름</label>
<input class="input input-md input-full">

<!-- Required -->
<label class="input-label">이메일 <span class="input-label-required">*</span></label>

<!-- Optional -->
<label class="input-label">닉네임 <span class="input-label-optional">(선택)</span></label>

<!-- Helper Text -->
<p class="input-helper">도움말 텍스트</p>

Inline Labels

레이블과 입력 필드를 가로로 배치하는 패턴입니다.

8자 이상 입력하세요

HTML 
<!-- Inline Layout -->
<div class="flex items-center gap-4">
  <label class="w-24 text-sm font-medium text-right flex-shrink-0">이름</label>
  <input class="input input-md flex-1 max-w-xs">
</div>

Floating Labels

입력 시 레이블이 위로 이동하는 플로팅 레이블 패턴입니다.

HTML 
<div class="input-floating-wrap">
  <input placeholder=" " class="peer input input-floating input-full">
  <label class="input-floating-label">레이블</label>
</div>

With Icons

아이콘이 포함된 입력 필드입니다. 입력 내용의 맥락을 시각적으로 전달합니다.

HTML 
<!-- Leading Icon -->
<div class="input-icon-wrap">
  <svg class="input-icon input-icon-left">...</svg>
  <input class="input input-md input-full pl-10">
</div>

<!-- Trailing Icon -->
<div class="input-icon-wrap">
  <input class="input input-md input-full pr-10">
  <svg class="input-icon input-icon-right">...</svg>
</div>

Input Groups

접두사, 접미사, 버튼 등이 결합된 입력 그룹입니다.

https://
@gmail.com
HTML 
<!-- Prefix Text -->
<div class="input-group">
  <span class="input-group-addon input-group-addon-left">https://</span>
  <input class="input input-md flex-1 input-rounded-l-none">
</div>

<!-- With Button -->
<div class="input-group">
  <input class="input input-md flex-1 input-rounded-r-none">
  <button class="btn btn-md btn-primary rounded-l-none">버튼</button>
</div>

<!-- With Select -->
<div class="input-group">
  <select class="input-group-select">...</select>
  <input class="input input-md flex-1 input-rounded-l-none">
</div>

Textarea

여러 줄의 텍스트를 입력받는 영역입니다.

0/200

HTML 
<!-- Default (resize-y) -->
<textarea rows="4" class="input textarea textarea-resize input-full"></textarea>

<!-- No Resize -->
<textarea rows="4" class="input textarea textarea-no-resize input-full"></textarea>

<!-- Disabled -->
<textarea rows="4" disabled class="input input-disabled textarea textarea-no-resize input-full"></textarea>

Usage Guidelines

                    

                        

                            

                                

                                    
                                    Do
                                

                                
    
                                    
  • • 항상 명확한 레이블을 함께 사용하세요
    • 
                                    
  • • for 속성으로 레이블과 입력 필드를 명시적 연결하세요
    • 
                                    
  • • 적절한 input type을 사용하여 모바일 키보드를 최적화하세요
    • 
                                    
  • • 필수 항목은 빨간 별표(*)로 표시하세요
    • 
                                    
  • • 에러 발생 시 구체적인 안내 메시지를 제공하세요
    • 
                                    
  • • 비밀번호 필드에는 보기/숨기기 토글을 제공하세요
    • 
                                    
  • • 충분한 터치 영역(44px 이상)을 확보하세요
    • 
                                

                            

                            

                                

                                    
                                    Don't
                                

                                
    
                                    
  • • placeholder를 레이블 대신 사용하지 마세요
    • 
                                    
  • • 레이블 없이 입력 필드만 배치하지 마세요
    • 
                                    
  • • 입력 중에 실시간 유효성 검사를 과도하게 하지 마세요
    • 
                                    
  • • 불필요한 필수 항목을 만들지 마세요
    • 
                                    
  • • 에러 메시지를 빨간색 테두리만으로 표시하지 마세요
    • 
                                    
  • • 레이블과 입력 필드를 너무 멀리 배치하지 마세요
    • 
                                    
  • • 비밀번호 입력에 자동완성을 막지 마세요
    •
